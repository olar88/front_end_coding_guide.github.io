Commit

Basic commit message


// [Header]
<Type> (Scope) : <Description> 
[Body] 
[Footers]
  1. Type : 本次提交的類型 (必)
    Type ENType ZHdefinition
    fix修正修正 bug
    feat新建修改 / 新增功能等
    docs文檔更動純文檔更動 ( 如: READ.md 等文件 )
    style調整整理 / 調整程式碼 ( 不影響程式主邏輯 ) ex: 註釋新增, 空格等排版調整
    refactor重構重構代碼 ( 無新功能或修復 bug ) ex: 改寫 TS
    perf優化改善效能
    build套件新建構建工具或外部依賴的更改 ( npm 等 )
    test測試增加測試 (when adding missing tests)
    chore維修(備份)除上述之外的修改 / 建構程序或輔助工具的變動
    revert撤銷撤銷先前的提交
  1. Scope : 本次改動的範圍 (選)

    填寫中 / 英文 改動檔案名稱

  1. Description : 本次改動的簡述 (必)

    簡短但精確地描述

  1. Body : 本次改動的完整描述 (選)

    詳細描述本次修改

  1. Footer : 備註事項 (選)

    是否為版本的變更或是 issue 的修復

//實例
fix UserLevelFirst: 職等搜尋參數修正
